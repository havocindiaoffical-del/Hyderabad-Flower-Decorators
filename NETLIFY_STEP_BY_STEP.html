Scroll down to the "Build settings" section. Click the Edit settings button (top-right of that section)

Find the "Publish directory" field. It currently shows something wrong (empty or the repo root). Change it to:

Click Save at the bottom of the form

✅ Fix 1 done! The publish directory should now show .next

2

Remove the Old Next.js Plugin (v4)

Your Netlify dashboard installed an old v4 plugin that's incompatible with Next.js 16. We need to remove it so the v5 plugin from your package.json takes over.

Step 1

Go back to your site's top navigation bar. Click on Integrations

Overview
Deploys
Site configuration
Integrations
Logs

↑ Click "Integrations" (highlighted in green)

Step 2

You'll see a list of installed integrations. Look for "Next.js" or "@netlify/plugin-nextjs". Click on it.

Next.js
@netlify/plugin-nextjs v4.41.6
Uninstall
Some other plugin
...

↑ Click the Next.js integration (highlighted in green)

Step 3

On the integration detail page, scroll down to the "Danger zone" section. Click Uninstall this extension (or Remove)

Next.js
@netlify/plugin-nextjs — Automatically configures Next.js on Netlify
⚠️ DANGER ZONE

Uninstalling will remove this integration from all sites using it.

Uninstall this extension

Step 4

Confirm the uninstall when prompted

⚠️ Can't find the integrations page? Try going to: https://app.netlify.com/teams/YOUR-TEAM/extensions?tab=installed (replace YOUR-TEAM with your team name)

⚠️ Still can't find it? Some users report that UI-installed plugins can't be removed through the dashboard. If that happens, go to Site configuration → Build & deploy → Continuous deployment → Build settings and check if there's a "Plugins" section where you can disable it.

✅ Fix 2 done! The old v4 plugin is removed. Your package.json's v5.15.12 will now be used automatically.

Add Resend API Key (Optional)

Only needed if you want email notifications when bookings come in. The site works without it.

Step 1

Step 2

Click Add a variable

Step 3

Key: RESEND_API_KEY    Value: re_your_key_here

Step 4

Click Save

🚀 Final Step: Deploy

  1. Click on the "Deploys" tab in the top navigation
  2. Click "Trigger deploy" button
  3. Select "Clear cache and deploy site"
  4. Wait for the build to complete (~2-3 minutes)
  5. You should see "Deploy is live!" 🎉
